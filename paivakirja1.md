# Oppimispäiväkirja: Paikallinen git

__Mikä osion tehtävissä oli vaikeaa ja mikä helppoa? Mikä auttoi minua oppimaan? Miten selvitin esteet?__

Aluksi unohdin käyttää git commit- komentoa aina git add - komennon jälkeen (vaikka ohjeissa niin oli), joten tehtävät menivät hieman sekaisin. Toki virheistä sitä eniten oppii. Aikaisemmilla kursseilla oli käyty läpi perustason liikkuminen komentorivillä. Joten se ei ollut täysin uutta. Toki käytän nyt MacBookkia niin jouduin hieman googlettelemaan komentoja.

Haarojen hahmottaminen oli aluksi hankalaa (en ymmärtänyt oikein graafeja), mutta lopulta ymmärsin graafit ja miksi haaroja tehdään ja, että niitä voidaan myös yhdistää. Uusien haarojen luominen vaikutti helpolta sekä niistä pois liikkuminen.

Suurempia esteitä ei oikeastaan ollut, tehtäväohjeet oli melko selkät, muutaman googlauksen jouduin tekemään. Liikuin paljon komentorivillä ja kokeilin kaikenlaista, joten se on ehdottomasti oikea tapa oppia tätä.



## Osiossa käyttämäni Git-komennot

| Komento | Kuvaus |
| --------| ------ |
| git clone | Kopioin olemassa olevan repositorion |
| git add | Lisään muutokset mukaan seuraavaan tallennukseen |
| git commit -m "versio 1.1 muutoksia"| Tallennus ja tekstiä mitä tehty |
| git status | Git tila, onko mitä muutoksia tehty yms. |
| git log | commit talletusten historian näkee |
| git rm | poistaa tiedoston |
| git mv  | nimesin esim HELLO -> hello.html|
| git revert  | talletuksen peruutus  |
| git switch | vaihdan haaraa|
| git merge | haarojen yhdistäminen|
