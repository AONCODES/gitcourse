# Oppimispäiväkirja: Git projektissa

__Mitä hyötyä voisi olla versionhallinnasta, jos kehität projektia yksin?__

Voi palata muihin versioihin. Voi tehdä eri kehityspolkuja ja kokeilla kaikenlaista ilman että päähaara rikkoutuu. Toimii myös varmuuskopiointina

__Mitä hyötyä voisi olla versionhallinnasta, jos projektissa on useita kehittäjiä?__

Käyttäjät voivat hakea etärepositoriosta muiden muutoksia ja myös itse viedä sinne omat muutokset. Työnjakaminen helpohkoa. Yhdistämispyynnöt mahdollistavat keskustelun ja projektin työstämisen usean kehittäjän kesken.

Keskitetyssä mallissa jokainen toimii paikallisessa repositoriossa ja ns käyttää yhteistä repositoriota jonne voi synkronoida työnsä. Tämä malli vaatii ryhmän jäsenille oikeudet viedä muutoksia yhteiseen repositorioon. Integrointimanagerimallissa oikeuksia on vähemmän ja muutosten vienti tapahtuu Integraatiomanagerin avulla.



__Miten järjestäisit projektitiimin versionhallinnan 3-4 hengen ohjelmistoprojektikurssilla? Laadi tiimiläisille lyhyt ohje, miten projektissa toimitaan.__



1. Kloonaa projekti - git clone <repositorio>
2. Tee muutokset aina omaan haaraan 
3. Talleta muutokset usein ja käytä selkeää max 50 merkin otsikkoa (mitä muutettiin)
4. Ennen puskemista hae muiden muutokset ja kun kaikki toimii lähetä omat muutokset
5. Tee ehdotettava muutos ominaisuushaaraan(merge request)
